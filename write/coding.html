Learning

My goal is to take you from 0 to website creation

This is meant to show you practical basics- nothing more

I use windows. There are some commands which may be different on a mac. Oops. If you are stuck- try Googling your problem

A few points before we dive in:

  • have confidence in yourself. you are smart enough. all you need to learn in a few things and then consitently apply them. you can do it
  • learning is need based so have a problem you want to solve. do not code just to code. solve a small, tiny, simple, problem
  • google stuff all the time. because 1) somebody has ran into your problem before and 2) you cannot know everything. even the best developers google!
  • computers are stupid but you are not. be precise. a computer will only do what you tell it
  • you can be faster at programming if you try. for everything that you do- there is always a quicker way. keep that in mind!
  • code along instead of using notes. it is tempting to write everything down- but your fingers have brains. code along and get used to writing in code instead of writing in pencil

Termianl

Every computer has a terminal. Search for the application- and see what comes up

Terminals are used to communicate with your computer via text, This is opposed to clicking around and using your mouse

Why use the terminal? For one it is versitile. Two, it is percise. Three, it is quicker.

As an example: consider driving somehwere vs Uber. Uber may be sexier- as you can use a sleek app from your phone. But driving your car is more versitile (Uber does not drive some places), more percise (Uber does not know exactly where you want to go) and faster (you have to wait for Uber)

Terminals are the weapon of choice for developers. There are not many commands you need to know, and it is easy to understand after a while

Note that a lot of these commands can be combined. Try them out!

Navigating the Terminal (for mac)
  • Tab | auto complete. if you have a file named myles.txt, type myl and the terminal will populate myles.txt
  • Ctrl + L | Clear screen. things get cluttered!
  • Shift + Cmd + V | paste plain text
  • Option + Shift + Cmd + C | Copy plain text
  • exit | exit terminal
Navigating Files
  • ls | lists files in the current directory. directory means folder. this will show your files, directories and more. Try it now
  • cd "directory" | change directory. this allows you to go in or out of directories. cd .. brings you back a directory. cd ~ bring you to the root directory (where your terminal starts. list your directories and cd into one. now cd back out
  • mkdir "new_directory" | creates a new directory. cd into downloads and create a new directory called "my_files"
  • touch "file_name"| creates a file. cd into your "downloads" directory and create a file named "abi.txt"
  • mv "file' "new_file" | move or rename file/directory. rename "abi.txt" to "yingyue.txt". now move your new file to "my_files" directory
  • rm "directory" -r | delete directory. now cd do your "downloads" directory, and delete "my_files"
Other Commands
  • start | start an application. start spotify
  • open | open a file. download any pdf. now open it.
  • vim/subl/code | open text editor (we will get to this later)

Terminal Quiz

Boom. That is it. you have mastered the terminal. Now all you need to do is practice!

navigate to your desktop. create a new directory "gabon_files". create a new dictory in this "gabon_files" named "terminal_practice". in "terminal_practice"- create one text file "terminal_quiz". now rename the text file to "passed_quiz". now move the file from "terminal_practice" to "gabon_files". now delete "terminal_practice". now using your browser, open up the text file. That is all you need to know for the terminal. Now you need to practice. Get used to navigating around your laptop using the commands we learned!

Environment

your "environment" is just what you use to code. you are already familiar with one part of your environment- the terminal. but you need a few more things for a full environment

besides your terminal- the text editor is the most important part of your environment. we will add more to your "environment" later

text editors range from minimal to robust. the truth is- nobody cares what text editor you use. all that matters is you can edit text and can program at a decent speed. Below are the 3 editors I have used extensively

  • Vim is the most hardcore editor. it was created in the 1970s, and acts like it. it comes pre-installed on your mac. if you become great at vim- you can be extremely fast. but, it takes forever to master vim. vim is speed. vim is pain. vim is life (or so they say).
  • Sublime Text is a light-weight editor. More extensive than Vim- this is my favorite. Not too much clutter. Smooth
  • Visual Studio Code is the most popular for full time developers who do not like pain (looking at you, Vim). This was created by Microsoft. It is robust. There are many plugins to make it even more robust- too

Again- nobody cares what text editor you use. They all get the job done. There are certainly pros and cons to each- but you should pick one and stick to it. Besides Vim- most text editors can be learned in an afternoon. Pick one. Learn the shortcuts. Stick to it. if you want my recommendation- use sublime text

Another terminal command! If you want to open a file with your code editor- then you can use vim/subl/code "file_name" to do so! If the file does not exist, then the file will be made and you can start editing right away! Try it now. Create a new text file with your text editor. save it. open it with your browswer.
